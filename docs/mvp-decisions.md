# 콕알림 MVP 결정 리스트

> 상태: 확정 (2026-07-07) — 대상: 외부인 4명 + 운영자
> 원칙: 심사·계약·UI가 필요한 것은 전부 배제. 발송 파이프라인과 DB는 확장 시 그대로 재사용.
> 배경 백로그: [mvp-serving-spec.md](mvp-serving-spec.md) / 충돌 시 이 문서가 우선

## 1. MVP 확정안 (4명용 — "나에게 보내기")

**구조: 카카오 developers 앱 1개 + 유저 팀원 등록 + '나에게 보내기' API. 채널·챗봇·알림톡·심사·딜러사 전부 없음.**

| # | 결정 | 채택안 | 비고 |
|---|---|---|---|
| 1 | 발송 수단 | **카카오 '나에게 보내기'** (카카오 로그인 talk_message) | 사업자·심사·딜러사·채널 불필요. 유저를 developers 앱 **팀원으로 등록**하면 검수 없이 사용 가능. 수신은 유저 본인의 "나와의 채팅"방 |
| 2 | MVP 범위 | **지정 시간 브리핑만** | 실시간 속보는 2차 |
| 3 | 등록 방식 | **수동** — 유저가 카톡으로 종목/수신시각을 알려주면 운영자가 DB 시드 | 등록 UI·챗봇 없음. 4명은 이게 최속 |
| 4 | 토큰 확보 | 유저에게 **동의 링크 1회 클릭** → 초소형 콜백 엔드포인트가 refresh token 저장 → 워커가 자동 갱신 | 콜백 1개만 공개 URL 필요 (Vercel 함수 등). 카카오 refresh token은 만료 전 자동 갱신으로 상시 유지 |
| 5 | 발송 주체·스케줄 | **연구실 PC 단독** — 수집 파이프라인 + 1분 폴링으로 `briefing_time` 도래 유저에게 발송 | outbound HTTP만. Vercel Cron·job 큐 불필요 |
| 6 | 스키마 보강 | `users.briefing_time` + `kakao_tokens(user_id, refresh_token, access_token, expires_at, …)` | 전화번호·알림톡 동의 필드 불필요. 하위호환 원칙(추가만, [db-schema.md](db-schema.md) §9) |
| 7 | 비용 | **0원** (LLM은 codex 구독 유지) | |

**감수하는 제약 (확장 트리거):**
- 수신이 채널방이 아니라 "나와의 채팅" — 브랜드 경험 없음
- 팀원 등록 방식이라 지인 수십 명이 한계 → **외부 공개·유저 확장 시점에 §2로 전환**

**할 일:**

```
[사장님 — 10분]                              [개발]
developers.kakao.com 앱 생성                 ① 스키마 보강(#6)
→ 카카오 로그인 ON + talk_message 동의항목    ② 워커 DB 전환 + 지정시간 브리핑 발송
→ 유저 4명 카카오계정 이메일 팀원 초대         ③ 동의 링크 콜백 엔드포인트 1개
→ 4명에게 종목/수신시각 수집                  (검증: 콘솔/텔레그램 채널로 파이프라인 선검증)
        ↘ 링크 클릭 4회 → 가동
```

## 2. 확장 트랙 (보류 — 유저를 늘리기로 결정한 시점에 개시)

**구조: 발송은 알림톡, 대화는 챗봇, 유저에겐 채널(@콕알림) 하나.** 용어 정의는 [mvp-serving-spec.md](mvp-serving-spec.md) 참고.

| # | 결정 | 채택안 | 비고 |
|---|---|---|---|
| E1 | 발송 수단 | 알림톡 | ⚠️ 하드 전제: **사업자등록번호** (비즈니스 인증·발신프로필 필수) |
| E2 | 딜러사 | 솔라피(Solapi) 우선 검토 | 비즈고·NHN도 무방 |
| E3 | 알림톡 템플릿 | 브리핑 1종: `[콕알림] {이름}님, {시각} 브리핑…` + BT 버튼(→챗봇) | 심사 리드타임 = 크리티컬 패스, 문구 확정 즉시 제출 |
| E4 | 전화번호 수집 | 챗봇 대화에서 직접 입력 + 동의 문구 | 알림톡은 전화번호 기반 |
| E5 | 챗봇 인텐트 | 종목 등록/해제/알림시간/내 설정/{종목} 오늘 소식 (5개) | 스킬 서버는 Vercel, DB 조회만, 5초 내 |
| E6 | 스키마 추가 | `users.phone`, `users.alimtalk_consent_at`, `deliveries.channel`에 `'alimtalk'` | MVP 스키마에 컬럼 추가만 |
| E7 | 비용 | 건당 ~8~10원 × 발송량 | 유저 수 확정 시 재산정 |

**확장 시 크리티컬 패스:** 채널 개설 → 비즈니스 인증 → 딜러사 가입 → 발신프로필 → 템플릿 심사 ∥ (개발) 발송 어댑터 교체 + 챗봇 스킬 서버. MVP의 파이프라인·DB·브리핑 로직은 전부 재사용, 바뀌는 건 발송 어댑터와 등록 경로뿐.
