> 제품 형태: **카카오채널(@콕알림) 하나 + 발송은 알림톡 + 대화(등록)는 챗봇.** 이 형태는 불변이며, 규모(4명이든 수천 명이든)에 따라 바뀌지 않는다.

## 크리티컬 패스

심사(비즈니스 인증 + 템플릿)만이 대기 시간이며, 개발을 막지 않는다.

```
[사장님 트랙 — 즉시 시작]                        [개발 트랙 — 병렬]
사업자 확인                                      ① 스키마 보강(#9)
→ 채널 개설(@콕알림)                             ② 워커 DB 전환(수집→events)
→ 비즈니스 인증 → 솔라피 가입(#2)                 ③ 브리핑 생성 + 지정시간 발송(#8)
→ 발신프로필 등록 → 템플릿 심사 제출(#4)              — 텔레그램으로 4명 선가동(#11)
→ 오픈빌더 봇 생성 + 채널 연결                    ④ 챗봇 스킬 서버(Vercel, #6·#7)
        ↘ 심사 통과 → 발송 어댑터 알림톡 교체 → 채널 정식 가동
```

- 개발 착수 순서: ① → ② → ③(텔레그램 선가동) → ④ → 알림톡 교체.
- 4명은 별도 구성이 아니라 **정식 파이프라인의 첫 수신자**다. 규모 때문에 제품 형태를 바꾸지 않는다.
