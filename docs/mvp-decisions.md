# 콕알림 MVP 결정 리스트 (카톡 알림톡+챗봇)

> 상태: 확정 대기 — #1(사업자 여부) 확인이 선결 조건
> 전제: 5~10명 유저, 장애대응·고가용성 배제, "지정한 시간에 카톡으로 받아보기"가 1차 목표
> 구조: **발송은 알림톡, 대화는 챗봇, 유저에겐 채널(@콕알림) 하나** — 용어·배경은 [mvp-serving-spec.md](mvp-serving-spec.md) 참고

## 결정 리스트

| # | 결정 | 채택안 | 비고 |
|---|---|---|---|
| 1 | 발송 수단 | **알림톡** | ⚠️ 유일한 하드 전제: **사업자등록번호** (채널 비즈니스 인증·발신프로필에 필수). 없으면 임시로 "나에게 보내기"(카카오 로그인 메시지 API, 유저를 개발자 앱 팀원 등록) 가동 후 사업자 확보 시 알림톡 전환하는 2단 구성 |
| 2 | 딜러사 | **솔라피(Solapi)** | 개인/소기업 가입 쉬움, API 문서 양호, 선불 소액충전. 비즈고·NHN도 무방 — 하나만 확정 |
| 3 | MVP 범위 | **지정 시간 브리핑만** | 실시간 속보는 2차 (템플릿 1장 추가로 켤 수 있음) |
| 4 | 알림톡 템플릿 | 브리핑 1종만 심사: `[콕알림] {이름}님, {시각} 브리핑\n관심 종목 {N}건의 새 소식\n- {헤드라인1}\n- {헤드라인2}…` + **BT 버튼**("자세히 보기" → 챗봇 전환) | 심사 리드타임 = 크리티컬 패스 → 문구 확정 후 딜러사 가입 즉시 제출 |
| 5 | 전화번호 수집 | **챗봇 대화에서 직접 입력** + 수집·이용 동의 문구 1줄 | 알림톡은 전화번호 기반, 챗봇이 번호를 자동 제공하지 않음. 카카오싱크 등 자동화 배제 |
| 6 | 챗봇 인텐트 | 최소 5개: **종목 등록 / 해제 / 알림시간 설정 / 내 설정 보기 / {종목} 오늘 소식** | 오픈빌더 블록 5개 + 스킬 서버 1개. 그 외 발화는 "도움말" 폴백 |
| 7 | 스킬 서버 위치 | **Vercel** — 공개 HTTPS, DB 읽기/쓰기만, 5초 내 응답, LLM 동기 호출 금지 | 얇은 핸들러(Node) 수십 줄 수준 |
| 8 | 발송 주체·스케줄 | **연구실 PC 단독** — 기존 수집 파이프라인 + 1분 폴링으로 `briefing_time` 도래 유저에게 솔라피 API 호출 | 발송은 outbound HTTP라 PC에서 가능. Vercel Cron·job 큐 불필요 |
| 9 | 스키마 보강 | `users.phone`, `users.alimtalk_consent_at`, `users.briefing_time` 추가 + `deliveries.channel`에 `'alimtalk'` 추가 | 하위호환 원칙(추가만, [db-schema.md](db-schema.md) §9). 챗봇 유저는 기존 `identities(provider='kakao')` |
| 10 | 비용 | 알림톡 건당 ~8~10원 → 10명 × 매일 1건 ≈ **월 3천원 미만**, LLM은 codex 구독 유지 | 속보 도입 시 건수 증가 → 그때 재확인 |

## 크리티컬 패스

심사(비즈니스 인증 + 템플릿)가 병목. 두 트랙을 병렬로 진행한다.

```
[사장님 트랙 — 오늘 시작]                        [개발 트랙 — 병렬]
사업자 확인(#1)                                  스키마 보강(#9)
→ 채널 개설(@콕알림)                             → 워커 DB 전환(수집→events)
→ 비즈니스 인증 → 솔라피 가입(#2)                → 브리핑 생성 + 발송 어댑터(alimtalk)
→ 발신프로필 등록 → 템플릿 심사 제출(#4)          → 챗봇 스킬 서버(Vercel, #6·#7)
→ 오픈빌더 봇 생성 + 채널 연결
        ↘ 심사 통과 시점에 양쪽 합류 → 유저 5~10명 초대 → 가동
```

- 심사 대기 중에도 개발·테스트는 전부 가능 — 텔레그램/콘솔 채널로 동일 파이프라인을 검증하고, 통과 후 어댑터만 알림톡으로 교체.
- 개발 트랙 착수 순서: #9 스키마 보강 → 워커 DB 전환 → 발송 어댑터 → 스킬 서버.
